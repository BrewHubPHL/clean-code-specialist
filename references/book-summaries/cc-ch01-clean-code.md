## Summary

Code will never disappear — even higher-level specifications remain formal, executable programs. Bad code compounds: productivity collapses, teams demand a "grand redesign in the sky," and the rewrite often fails for the same reasons. Clean code is not aesthetic preference; it is professional survival. The chapter defines clean code through multiple expert lenses (Bjarne Stroustrup, Grady Booch, Dave Thomas, Michael Feathers, Ron Jeffries, Ward Cunningham) and lands on the **Boy Scout Rule**: leave the campground cleaner than you found it.

## Actionable rules

1. Treat clean code as a professional obligation — no reasonable excuse for shipping less than your best.
2. Never defer cleanup with "later" — **LeBlanc's law**: later equals never.
3. Apply the **Boy Scout Rule** on every touch: one small improvement per edit beats heroic rewrites.
4. Read code as an author writes prose — clarity, rhythm, and intent for the next reader.
5. When productivity drops on a mature codebase, suspect accumulating mess before blaming headcount.
